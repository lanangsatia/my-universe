🔒 Pembayaran diproses oleh Midtrans

Alur Pembayaran Lengkap

  1. Klik "Terbitkan Globe ✨" → sistem panggil POST /api/payment/qris
  2. Midtrans Snap transaction dibuat → payment record PENDING
  3. Muncul modal "Bayar untuk Terbitkan" dengan total Rp 29.999
  4. Klik "💳 Bayar Sekarang" → popup Snap dari Midtrans muncul

Metode Pembayaran yang Didukung (Midtrans Snap)

QRIS GoPay OVO Bank Transfer lainnya

🔄 Polling Status

Setelah membayar, frontend secara otomatis:

  • Memanggil GET /api/payment/status/[orderId] setiap 3 detik
  • Jika status berubah PAID → otomatis lanjut ke proses publish
  • Timeout setelah 5 menit

🔄 Webhook Midtrans

Midtrans juga mengirim notifikasi ke server via POST /api/payment/webhook:

  • transaction_status = settlement atau capture → status diupdate ke PAID
  • Redirect callback: /payment/[orderId]?status=success|failed|pending
🧪 Mode Sandbox: Jika MIDTRANS_IS_PRODUCTION=false, tombol "Simulasi Bayar ✅" muncul untuk testing tanpa bayar sungguhan. Klik → langsung mark PAID.
5

Globe Live!

Setelah pembayaran sukses, sistem otomatis memanggil POST /api/globe/publish:

  • Foto diupload ke Cloudflare R2 (atau local fallback)
  • Data user, foto, dan konfigurasi disimpan ke PostgreSQL via Prisma
  • Globe langsung live di link unik: /u/[slug]

Detail Teknis API

Endpoint API yang terlibat dalam flow transaksi:

1. Create Midtrans Transaction

EndpointPOST /api/payment/qris
Auth✅ Required (Clerk session)
Body{ "amount": 29999 }
Response{ "reference_id": "GLOBE-...", "token": "...", "redirect_url": "..." }
FungsiBuat transaksi Midtrans Snap, simpan payment record PENDING

2. Check Payment Status

EndpointGET /api/payment/status/[orderId]
Response{ "status": "PENDING" | "PAID" }
FungsiCek status pembayaran dari database (polling tiap 3 detik)

3. Simulasi Bayar (Sandbox)

EndpointPOST /api/payment/simulate
Body{ "order_id": "GLOBE-..." }
FungsiMark payment sebagai PAID (hanya development/sandbox)

4. Publish Globe

EndpointPOST /api/globe/publish
Auth✅ Required (Clerk session)
BodyFormData — title, slug, photos (File[])
FungsiUpload foto ke R2, create/update user, return URL globe

5. Webhook Midtrans

EndpointPOST /api/payment/webhook
FungsiTerima notifikasi Midtrans. settlement/capture → PAID

6. Save Settings

EndpointPUT /api/globe/settings
Body{ "config": { ... } }
FungsiSimpan konfigurasi globe (tema, teks, kecepatan, meteor)

Struktur Database

Flow transaksi melibatkan 3 model database (PostgreSQL + Prisma):

User (table: users)

FieldTipeDeskripsi
idUUIDPrimary key
clerkIdString?Clerk user ID (unique)
slugStringURL slug (unique)
packageString"free" | "basic" | "premium"
maxPhotosIntMax foto (default 5)
configJsonTheme, text, speed settings

Payment (table: payments)

FieldTipeDeskripsi
orderIdStringOrder ID Midtrans (unique)
amountIntRp 29.999
statusString"PENDING" → "PAID"
paidAtDateTime?Waktu pembayaran sukses

Photo (table: photos)

FieldTipeDeskripsi
imageUrlTextURL foto (R2 atau local fallback)
userIdUUIDForeign key → User
📌

Catatan Penting

  • 🧪 Mode Sandbox: MIDTRANS_IS_PRODUCTION=false → tombol "Simulasi Bayar ✅" muncul untuk testing tanpa bayar
  • 🔧 Mock Mode: Jika MIDTRANS_SERVER_KEY kosong, sistem otomatis mock — token palsu, redirect lokal
  • ⏱ Polling: Frontend cek status tiap 3 detik, timeout 5 menit
  • 🌐 Webhook: Midtrans kirim notifikasi ke /api/payment/webhook (endpoint publik)
  • 💾 R2 Fallback: Jika R2 tidak dikonfigurasi, foto disimpan di public/uploads/
  • 🔒 Keamanan: Semua endpoint admin & dashboard diproteksi Clerk middleware
